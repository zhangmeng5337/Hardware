# Contributing Guidelines

## 目的
本文件定义本仓库的贡献准则与开发偏好，帮助保持稳定性、可维护性与一致的代码风格。

## 总体原则
- 在后续开发中尽量不改动原有代码（"最小侵入原则"）。优先使用以下策略：
  - 通过新增类、方法、扩展方法或辅助文件实现新功能。
  - 使用依赖注入、包装器或适配器模式将新行为与旧实现解耦。
  - 对 UI 或配置相关变更，优先新增资源/控件而不是修改已有控件的行为。
- 仅在确有必要且收益明显时对现有代码进行修改。必须在 pull request 中清晰说明：
  - 修改的文件与行数范围。
  - 修改原因与预期改进。
  - 潜在风险与回归测试计划。

## 变更流程
1. 提交 issue 讨论设计与替代方案。短小变更可直接在 PR 中描述并引用相关 issue。
2. 在 PR 描述中列出修改点、风险评估和回归测试步骤。
3. 至少一位审查者通过审查后方可合并。

## 编码风格与格式
- 遵守仓库中现有的 `.editorconfig` 配置（如果存在）。
- 命名、缩进、注释风格应与现有代码一致。

## 单元测试
- 重要逻辑变更应附带单元测试或手动回归测试说明。

## 文档
- 对公共 API 或行为变更请补充相应文档或注释。

## 例外情况
- 若必须进行侵入性改动，应在 PR 中提供迁移指南和回滚计划。

---

遵循以上准则可以减少回归风险，便于团队协作与代码维护。
